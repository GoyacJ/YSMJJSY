# /chat 星空剧场设计规格

日期：2026-05-16

## 背景

当前 `/chat` 已有多钥匙空间、流式对话、媒体输入、媒体生成、聊天记录保存、复制和下载能力。页面的问题不在功能数量，而在第一眼的记忆点不足。用户进入后还像“带星空背景的聊天页”，没有充分形成“这是一片只属于她的星空”的感受。

本规格只定义 `/chat` 视觉和交互升级，不改变 MiniMax 接入方式，不改变钥匙体系，不改认证模型。

## 参考来源

- Pretext：作为文本布局和流式文字稳定渲染参考。重点不是 UI 皮肤，而是文本测量、滚动锚定、逐字排版和长列表稳定性。
- Vercel Chatbot：参考 artifacts 思路。生成内容不只作为普通消息，而是成为可查看、可下载、可再次使用的内容对象。
- assistant-ui：参考附件、composer、thread action 的组织方式。
- Open WebUI：参考多模态能力和会话持久化思路，但不采用后台工具型视觉。
- run-llama/chat-ui：参考轻量聊天体验和多模态输入组织。

## 设计目标

`/chat` 不表现为普通聊天软件。它应该像一个“星空剧场”：

用户把文字、声音、图片或视频投进星空。星空以文字、语音、图像、音乐或视频回信。每个钥匙进入的是不同空间，记忆和内容都属于当前钥匙。

视觉关键词：

- 温柔
- 安静
- 星空
- 仪式感
- 私人感
- 高级但不喧闹

## 页面结构

页面分三层。

### 背景层

背景层负责氛围，不承载主要交互。

内容：

- 动态星空
- 慢速星尘
- 少量流星
- 根据当前空间情绪变化的色彩
- 设计模式下增加轻微网格或星图辅助线

要求：

- 不使用大面积单色渐变。
- 不使用装饰性彩色圆球。
- 背景动效不能影响输入和阅读。
- 移动端降低粒子数量。

### 内容层

内容层承载聊天、媒体结果和记忆状态。

包括：

- 星信消息流
- 媒体星体卡片
- 记忆星图入口
- 当前空间状态提示

消息不做重边框卡片。默认像浮在星空中的文字。用户触摸、点击或悬浮后，消息才显示操作按钮。

### 控制层

控制层是底部输入控制台。

包括：

- 附件按钮
- 语音输入按钮
- 设计模式按钮
- 媒体意图按钮：听一听、画一张、做一段、写一首
- 输入框
- 发送按钮

控制层要稳定在底部。高度比当前更克制。默认状态安静，聚焦时有轻微呼吸动画和星光反馈。

## 核心体验

### 星信消息

消息显示方式：

- 用户消息：像投递出去的短笺，靠右或略偏右。
- AI 消息：像星轨文字浮现，靠左或自然铺开。
- 系统状态：用低对比状态线表示，不占主要视觉。

交互：

- 悬浮、点击、触摸消息时显示复制按钮。
- 媒体消息显示下载按钮。
- 新消息自动滚动到最新。
- 读取历史时保留媒体内容。

流式输出：

- 回复继续逐字浮现。
- Pretext 可用于后续优化文本测量、换行和滚动稳定性。
- 如果暂不引入 Pretext，先保留现有 Vue 渲染，但 CSS 和结构要为替换文本渲染层留接口。

### 媒体星体

媒体结果不塞进普通文字气泡。

图片：

- 以发光星体卡片展示。
- 缩略图在消息中。
- 点击进入沉浸预览。
- 支持下载。

音频和音乐：

- 用波形轨道展示。
- 播放控件保持可访问。
- 支持下载。

视频：

- 视频功能目前不作为重点。
- 已有返回时用任务卡片或视频缩略卡片展示。
- 不在本阶段重做生成链路。

### 输入控制台

底部输入框称为“星信控制台”。

默认聊天模式：

- placeholder 使用自然表达，不写说明书式文案。
- 回车发送，Shift + 回车换行。
- 附件、语音、媒体意图都在输入框内部组织。

设计模式：

- 选中设计图标后，控制台进入设计状态。
- 输入内容被视为页面设计指令。
- 控制台边缘、背景和按钮状态变化。
- 发送后触发当前已有的 `designRequested` 流程。

媒体意图：

- 点击“听一听 / 画一张 / 做一段 / 写一首”只是选择意图。
- 发送时只走一条星信请求。
- AI 回复仍回到同一条消息流。
- 音频意图遵循“星信先回复，再把回复转成语音”。

### 记忆星图

记忆和配置不应只像设置表单。

新增一个轻量“记忆星图”入口：

- 模型称呼是主星。
- MBTI 是星座形状。
- 近期情绪影响星空色调。
- 重要记忆作为小星点。

点击后打开设置面板。现有 `ProfileSettingsSheet` 可继续承载编辑能力，入口和视觉表达升级。

## 组件边界

建议拆分为这些组件：

- `StarChat.vue`：保留容器和状态编排。
- `StarChatThread.vue`：消息列表、自动滚动、消息激活。
- `StarChatMessage.vue`：单条消息、复制、媒体渲染入口。
- `StarMediaCard.vue`：图片、音频、音乐、视频展示和下载。
- `StarComposer.vue`：输入框、附件、语音、媒体意图、发送。
- `StarMemoryMap.vue`：记忆星图入口和状态展示。

拆分目标是降低 `StarChat.vue` 复杂度，不改变外部事件：

- `initialMessages`
- `sendMessageStream`
- `designRequested`

## 数据流

进入 `/chat`：

1. `pages/chat.vue` 加载当前钥匙设计。
2. 加载当前钥匙聊天历史。
3. 渲染背景层和星信层。
4. 输入控制台根据模式发送请求。

发送聊天：

1. `StarComposer` 产生 payload。
2. `StarChat` 追加用户消息。
3. `useStarChat.sendMessageStream` 发送到 `/api/chat/stream`。
4. 流式事件更新 assistant 消息。
5. 媒体结果进入同一条消息。

设计模式：

1. `StarComposer` 产生设计指令。
2. `StarChat` 触发 `designRequested`。
3. `pages/chat.vue` 预览设计。
4. 用户确认后提交设计。

## 状态和错误

错误展示要保持轻，不打断氛围。

- 附件类型错误：显示在输入控制台上方。
- 请求失败：保留用户消息，显示一条低对比错误状态。
- 语音输入不可用：按钮显示不可用态，并在附近显示短提示。
- 媒体下载失败：不提前拦截。URL 下载交给浏览器处理。

## 响应式

桌面端：

- 消息层最多占页面中下区域。
- 右上角额度显示降噪，不抢视觉。
- 记忆星图入口放在右侧或右下角。

移动端：

- 输入控制台宽度接近全屏。
- 工具按钮可以横向滚动。
- 消息显示更集中。
- 背景粒子减少。

## 可访问性

- 所有图标按钮保留 `aria-label`。
- 音频和视频使用原生 controls。
- 复制、下载按钮支持键盘聚焦。
- 对话区域保留 `aria-live="polite"`。
- 深色和浅色主题文字对比必须可读。

## 不做的事

本阶段不做：

- 重写 MiniMax 服务层。
- 新增登录系统。
- 新增服务器端多租户权限模型。
- 重做视频生成链路。
- 引入复杂 3D 引擎。
- 把页面做成后台管理工具。

## 验收标准

- `/chat` 首屏一眼能看出“私人星空剧场”。
- 底部输入控制台更轻、更稳定。
- 发送文本、语音输入、上传图片、上传音频、上传视频入口仍可用。
- 媒体意图仍通过同一条星信请求发送。
- 历史消息和媒体内容刷新后仍可恢复。
- 深色和浅色模式下消息、输入框、弹框都可读。
- 桌面和移动端没有明显重叠。
- 现有单元测试、构建和 E2E 测试通过。
